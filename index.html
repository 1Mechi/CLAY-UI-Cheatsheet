Enum Values
CLAY( id, { declaration } ) { children... } macro
The top-level macro. First arg is an element ID (from CLAY_ID). Second arg is a Clay_ElementDeclaration struct with all the options below. Children are declared inside the braces — this is the famous "for-loop trick".
id Clay_ElementId
The unique ID for this element. Used for hover detection, scroll lookup, transitions, and debug. Always the FIRST argument to CLAY().
CLAY_ID("MyName") macro → Clay_ElementId
Hashes a string literal into a stable uint32 ID. Use this for KNOWN, FIXED names. The string must be a literal — "MyName", not a variable. Why literal? Because CLAY_STRING() checks at compile time.
CLAY_IDI("Name", index) macro → Clay_ElementId
Like CLAY_ID but adds a number. Perfect for lists: CLAY_IDI("Item", i) makes "Item0", "Item1", etc. without building a dynamic string.
CLAY_SID(clay_string) macro → Clay_ElementId
Like CLAY_ID but accepts a Clay_String (dynamic, runtime string). Use when the name is built at runtime (e.g. from a std::string).
CLAY_ID_LOCAL("Name") macro → Clay_ElementId
Like CLAY_ID but scoped to the PARENT element's ID. Two children in different parents can both use CLAY_ID_LOCAL("Row") safely.
.layout Clay_LayoutConfig
Controls HOW children are arranged inside this element. Sizing, padding, gaps, direction, alignment all live here. Think of it like CSS flexbox config.
.sizing Clay_Sizing
How big is THIS element? Contains .width and .height, each a Clay_SizingAxis.
.width / .height Clay_SizingAxis
Each axis has a TYPE (how to size) and an optional min/max range.
CLAY_SIZING_FIXED(px) exact pixels
Element will always be exactly this many pixels wide/tall. Like CSS: width: 200px; Example: .sizing = { .width = CLAY_SIZING_FIXED(200) }
CLAY_SIZING_GROW(min?, max?) fills parent
Expands to fill remaining space in the parent, shared equally with other GROW siblings. Like CSS: flex: 1; Optional min/max constrain the growth range. Example: CLAY_SIZING_GROW(0) — grow with no minimum.
CLAY_SIZING_FIT(min?, max?) wraps content
Shrinks/grows to exactly fit its children. Like CSS: width: fit-content; Optional min/max prevent it getting too small or too big. Default behaviour when you don't set sizing.
CLAY_SIZING_PERCENT(0.0–1.0) % of parent
Percentage of the PARENT's size on this axis. 0.5 = 50%, 1.0 = 100%. NEVER pass 50 — pass 0.5. Like CSS: width: 50%;
.padding Clay_Padding { left, right, top, bottom }
Empty space INSIDE this element, between its edges and its children. All values are uint16 (pixels). Shortcut: CLAY_PADDING_ALL(8) sets all four sides to 8px. Like CSS padding.
.childGap uint16 (pixels)
Gap BETWEEN each child element along the layout axis. Like CSS gap in a flex container. Example: .childGap = 12 (12px between every child)
.layoutDirection Clay_LayoutDirection enum
Which direction do children stack? Defaults to LEFT_TO_RIGHT (horizontal).
CLAY_LEFT_TO_RIGHT default
Children placed side by side, left → right. Like flex-direction: row.
CLAY_TOP_TO_BOTTOM
Children stacked top → bottom. Like flex-direction: column.
.childAlignment Clay_ChildAlignment { x, y }
How children are aligned on EACH axis inside the parent's content area (after padding is subtracted). Has .x and .y sub-fields. Like CSS justify-content + align-items.
.x → CLAY_ALIGN_X_LEFT / _CENTER / _RIGHT
Horizontal alignment of children. LEFT is default.
.y → CLAY_ALIGN_Y_TOP / _CENTER / _BOTTOM
Vertical alignment of children. TOP is default.
.backgroundColor Clay_Color { r, g, b, a }
Solid fill color for this element. Values 0–255 by convention. If ONLY backgroundColor is set (no image/custom), Clay emits a RECTANGLE render command. Alpha of 0 = transparent = no rectangle drawn. Example: .backgroundColor = { 255, 100, 50, 255 }
.overlayColor Clay_Color { r, g, b, a }
Tints THIS element AND all its children with a color blend. The alpha controls blend strength: 0 = no tint, 255 = fully overlaid. Emits OVERLAY_COLOR_START / OVERLAY_COLOR_END render commands. Like CSS filter: brightness() but color-tinted.
.cornerRadius Clay_CornerRadius { topLeft, topRight, bottomLeft, bottomRight }
Round the corners. Each corner is a separate float (pixel radius). Shortcut: CLAY_CORNER_RADIUS(8) sets all four corners to 8px. Like CSS border-radius.
.aspectRatio Clay_AspectRatioElementConfig { aspectRatio }
Force this element to maintain a width/height ratio. aspectRatio = width / height. E.g. 16.0f/9.0f for widescreen. Fix one axis with CLAY_SIZING_FIXED and let Clay compute the other.
.image Clay_ImageElementConfig { imageData }
Makes this element render as an image. .imageData is a void* — pass a pointer to your Texture2D (raylib). The renderer receives it and calls DrawTexturePro(). Emits an IMAGE render command.
.floating Clay_FloatingElementConfig
Makes this element "float" — it layers over other elements (like CSS position:absolute) and does NOT push siblings around. You MUST set .attachTo to something other than NONE for floating to activate.
.attachTo Clay_FloatingAttachToElement
WHERE to attach this floating element. Must be set or floating does nothing.
CLAY_ATTACH_TO_NONE default — disables floating
CLAY_ATTACH_TO_PARENT relative to direct parent
CLAY_ATTACH_TO_ELEMENT_WITH_ID relative to .parentId
CLAY_ATTACH_TO_ROOT absolute position on screen
.attachPoints Clay_FloatingAttachPoints { element, parent }
Which corner/edge of ELEMENT aligns to which corner/edge of PARENT. 9 options each: LEFT_TOP, LEFT_CENTER, LEFT_BOTTOM, CENTER_TOP, CENTER_CENTER, CENTER_BOTTOM, RIGHT_TOP, RIGHT_CENTER, RIGHT_BOTTOM.
.offset Clay_Vector2 { x, y }
Pixel offset AFTER the attach point is calculated. Fine-tuning position.
.expand Clay_Dimensions { width, height }
Grow the bounding box outward without moving children. Useful for hit areas.
.parentId uint32 — from CLAY_ID().id
When attachTo = CLAY_ATTACH_TO_ELEMENT_WITH_ID, set this to the target's ID. Example: .parentId = CLAY_ID("Sidebar").id
.zIndex int16 — draw order
Higher zIndex = drawn on top. All children of a floating element inherit its zIndex. Non-floating elements are always zIndex 0.
.pointerCaptureMode Clay_PointerCaptureMode
Does this floating element "eat" mouse clicks or let them pass through?
CLAY_POINTER_CAPTURE_MODE_CAPTURE default — blocks clicks
CLAY_POINTER_CAPTURE_MODE_PASSTHROUGH clicks go through
.clipTo Clay_FloatingClipToElement
Should this floating element be clipped by the same scissor rect as its parent?
CLAY_CLIP_TO_NONE default — no clipping
CLAY_CLIP_TO_ATTACHED_PARENT
.custom Clay_CustomElementConfig { customData }
Pass ANY pointer to the renderer as a CUSTOM render command. In the renderer, cast customData back to your struct (e.g. CustomLayoutElement*). Used for 3D models, video, canvas, etc. — anything Clay can't render itself.
.clip Clay_ClipElementConfig { horizontal, vertical, childOffset }
Clips (hides) content that overflows this element's bounds — creates a scroll container. Set .horizontal = true and/or .vertical = true to enable clipping on those axes. .childOffset is a Clay_Vector2 used to scroll children (managed by Clay_UpdateScrollContainers). Emits SCISSOR_START / SCISSOR_END render commands.
.horizontal bool — clip on X axis
.vertical bool — clip on Y axis
.childOffset Clay_Vector2 — scroll position
The scroll offset. Pair with Clay_GetScrollOffset() inside the element declaration to wire in Clay's built-in scroll tracking. Example: .childOffset = Clay_GetScrollOffset()
.border Clay_BorderElementConfig { color, width }
Draws a border INSIDE the bounding box of this element (inset border). Has .color (Clay_Color) and .width (Clay_BorderWidth with left/right/top/bottom/betweenChildren). CLAY_BORDER_ALL(2) sets all four sides to 2px wide.
.color Clay_Color
Single color for all borders on this element.
.width Clay_BorderWidth { left, right, top, bottom, betweenChildren }
Individual side widths in pixels (uint16). .betweenChildren draws lines BETWEEN child elements (like a divider). Uses the layout direction to know if dividers are vertical or horizontal.
.transition Clay_TransitionElementConfig
Animate changes to this element (position, size, color, etc). Uses a callback function you provide (or the built-in Clay_EaseOut). Must set .handler and .duration at minimum.
.handler function pointer — bool fn(Clay_TransitionCallbackArguments)
Called every frame during a transition. Mutate arguments.current to set the interpolated state. Return true when the transition is complete. Use the built-in: .handler = Clay_EaseOut
.duration float (seconds)
How long the transition lasts. E.g. 0.3f = 300ms.
.properties Clay_TransitionProperty (bitmask)
WHICH properties to animate. Combine with | operator: CLAY_TRANSITION_PROPERTY_X | CLAY_TRANSITION_PROPERTY_Y Options: X, Y, WIDTH, HEIGHT, BACKGROUND_COLOR, OVERLAY_COLOR, BORDER_COLOR, BORDER_WIDTH Or use combined: POSITION, DIMENSIONS, BOUNDING_BOX, BORDER
.enter { setInitialState, trigger }
Config for ENTER animation (when element first appears). .setInitialState is a function that returns the starting state (e.g. start transparent, start off-screen). .trigger controls whether parent-on-first-frame triggers it.
.exit { setFinalState, trigger, siblingOrdering }
Config for EXIT animation (when element is REMOVED from layout). .setFinalState returns where to animate TO (e.g. fade out, slide out). Without this, removed elements just disappear instantly. .siblingOrdering controls whether the exiting element renders under/over siblings.
CLAY_TEXT(text, config) macro — leaf node, no children
NOT a child element declared with CLAY() — it's a separate macro. Call it INSIDE a CLAY() block. It creates a text leaf node. text is a Clay_String (use CLAY_STRING("hello") for literals). config is a Clay_TextElementConfig.
.textColor Clay_Color
Color of the text. RGB 0–255.
.fontId uint16 — index into your fonts array
Which font to use. You define the mapping — Clay just passes this number to your MeasureText function and to the renderer. fontId=0 is the debug view's font. Keep 0 as your default font.
.fontSize uint16 — pixels
.letterSpacing uint16 — extra px between chars
.lineHeight uint16 — 0 = auto
.wrapMode Clay_TextElementConfigWrapMode
CLAY_TEXT_WRAP_WORDS — break on spaces (default) CLAY_TEXT_WRAP_NEWLINES — only break on \n CLAY_TEXT_WRAP_NONE — never wrap
.textAlignment Clay_TextAlignment
CLAY_TEXT_ALIGN_LEFT (default) CLAY_TEXT_ALIGN_CENTER CLAY_TEXT_ALIGN_RIGHT Aligns wrapped lines within the text bounding box.
